This guide provides step-by-step instructions for setting up a local development environment for LoanFlow, a multi-step loan application platform built with React, TypeScript, and Vite.

## Prerequisites

Before starting, ensure you have the following installed on your system:

- **Node.js**: Version 18.x or higher (the project uses ES modules and modern JavaScript features)
- **npm**: Version 9.x or higher (comes bundled with Node.js)
- **Git**: For cloning the repository

You can verify your installations by running:

```bash
node --version
npm --version
```

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/aqureshiest/loanflow.git
cd loanflow
```

### 2. Install Dependencies

The project uses npm as its package manager. Install all required dependencies:

```bash
npm install
```

This installs:
- **React 18.2.0** and React DOM for the UI framework
- **TypeScript 5.2.2** for type safety
- **Vite 5.0.8** as the build tool and development server
- **Tailwind CSS 3.4.1** for styling
- **Radix UI components** for accessible, unstyled UI primitives
- **React Router DOM 6.21.1** for client-side routing
- **ESLint** and TypeScript ESLint for code quality

## Starting the Development Server

To start the development server with hot module replacement:

```bash
npm run dev
```

The application will be available at `http://localhost:5173` (or the next available port if 5173 is in use). Vite will display the exact URL in your terminal.

## Development Workflow

### Hot Module Replacement (HMR)

Vite provides fast, reliable hot module replacement out of the box. When you modify any `.tsx`, `.ts`, or `.css` file, the changes are reflected in your browser instantly without a full page reload. This applies to:

- React component changes
- Style modifications (Tailwind CSS)
- Route definitions
- State management updates

### Common Development Tasks

#### Running the Linter

Check your code for style violations and potential issues:

```bash
npm run lint
```

This runs ESLint with TypeScript support and enforces zero warnings. The configuration includes:
- React Hooks rules
- React Refresh rules
- TypeScript best practices

Fix automatically fixable issues with:

```bash
npm run lint -- --fix
```

#### Building for Production

Create an optimized production build:

```bash
npm run build
```

This command:
1. Runs TypeScript compiler (`tsc`) to check for type errors
2. Bundles and minifies the application with Vite
3. Outputs to the `dist/` directory

#### Previewing the Production Build

After building, preview the production bundle locally:

```bash
npm run preview
```

This serves the `dist/` directory and helps verify that the production build works correctly before deployment.

## Project Structure

The development environment is configured for the following structure:

- **`src/`** – Source code (React components, pages, utilities)
- **`src/pages/`** – Page components for each route (e.g., `RefinanceAmountPage.tsx`, `ReviewPage.tsx`)
- **`src/components/`** – Reusable UI components
- **`src/components/ui/`** – Shadcn/Radix UI component wrappers
- **`dist/`** – Production build output (generated by `npm run build`)
- **`index.html`** – Entry HTML file
- **`tailwind.config.js`** – Tailwind CSS configuration
- **`tsconfig.json`** – TypeScript configuration
- **`vite.config.ts`** – Vite configuration

## Configuration Files

### TypeScript Configuration

The project uses strict TypeScript settings. Key configurations:

- **Target**: ES2020 (modern JavaScript features)
- **Module**: ESNext (ES modules)
- **JSX**: React-JSX (automatic JSX transform)
- **Strict mode**: Enabled for type safety

### Tailwind CSS

Tailwind is configured with:

- **Base color**: Slate
- **CSS variables**: Enabled for theming
- **Plugins**: `tailwindcss-animate` for animation utilities
- **CSS file**: `src/index.css`

### Path Aliases

The project uses path aliases for cleaner imports:

- `@/components` → `src/components`
- `@/lib/utils` → `src/lib/utils`

Use these in imports instead of relative paths:

```typescript
// ✓ Preferred
import { Button } from "@/components/ui/button";

// ✗ Avoid
import { Button } from "../../../components/ui/button";
```

## Troubleshooting

### Port Already in Use

If port 5173 is already in use, Vite will automatically use the next available port. Check the terminal output for the actual URL.

To specify a custom port:

```bash
npm run dev -- --port 3000
```

### Module Not Found Errors

Ensure all dependencies are installed:

```bash
npm install
```

Clear the node_modules cache and reinstall if issues persist:

```bash
rm -rf node_modules package-lock.json
npm install
```

### TypeScript Errors

Run the TypeScript compiler to check for type errors:

```bash
npx tsc --noEmit
```

Fix any reported type issues before building.

### Linting Failures

The build process enforces zero ESLint warnings. Run the linter to identify issues:

```bash
npm run lint
```

Use `--fix` to automatically resolve fixable issues:

```bash
npm run lint -- --fix
```

### Hot Module Replacement Not Working

If HMR is not updating your browser:

1. Check that the development server is running (`npm run dev`)
2. Verify your file changes are saved
3. Check the browser console for errors
4. Restart the development server if issues persist

### Build Failures

If `npm run build` fails:

1. Check for TypeScript errors: `npx tsc --noEmit`
2. Check for linting errors: `npm run lint`
3. Ensure all dependencies are installed: `npm install`
4. Clear the build cache: `rm -rf dist/`
